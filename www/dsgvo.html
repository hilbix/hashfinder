DSGVO

Hinweise zum Datenschutz.

  • Die Übermittlung personenbezogener Daten an diese Webseite ist verboten!
  • Diese Webseite erhebt, verarbeitet und speichert von sich aus keinerlei personenbezogenen Daten, außer jemand verstößt gegen den vorherigen Punkt.
  • Ein Verstoß gegen diese Bestimmung stellt einen Missbrauch dieses Systems dar.
  • In diesem Fall ist derjenige für die Einhaltung der DSGVO verantwortlich, der das System missbraucht hat!
  • Gem. Kurt Gödel (Mathematiker) kann es kein Verfahren geben, das einen Missbrauch erkennen oder unterbinden könnte.
  • Alle Daten dieser Webseite und alle Daten, die an diese Website zur Speicherung übermittelt werden, sind auf Dauer frei und öffentlich zugänglich.
  • Dieses System basiert technisch notwendigerweise auf einer Blockchain. Eine Daten-Löschung ist deshalb technisch nicht vorgesehen, da dies die Integrität der Blockchain zerstört!
  • Sofern Sie ein Problem mit irgendwelchen auf dieser Webseite gespeicherten Daten haben, klicken Sie bitte auf den "Report"- oder "Block"-Button neben den Daten.
    • Der "Report"-Button meldet die Daten zur Überprüfung an den Betreiber.
    • Der "Block"-Button meldet die Daten zur Überprüfung an den Betreiber und sperrt sofort den weiteren Zugang zu den Daten.
    • In dem dann erscheinenden Textfeld können einen erklärenden Text hinzufügen. Dies ist aber nicht erforderlich, die Meldung ist dann bereits erfolgt.
    • Ist der "Block"-Button ausgegraut, wurden die Daten bereits vom Betreiber geprüft und als problemlos eingestuft. In diesem Fall hat eine Meldung per "Report" ohne erklärenden Text wenig Zweck und wird ignoriert.
  • Diese Webseite erhebt, verarbeitet und speichert pseudonyme Daten, z. B. die IP-Adresse
  • Die pseudonymen Daten dienen der Nachforschung, dem technischen Logging sowie dem Schutz dieser Webseite.
  • Eine Weitergabe der pseudonymen Daten an Dritte findet nicht statt.
  • Die Pesudonymen Daten sind so lange pseudonym, bis sie entweder durch die Eingabe personenbezogener Daten durch die IP selbst oder durch den ISP dem diese IP zugeordnet wurde depseudonymisiert werden.
  • Abgesehen davon ist eine depseudonymisierung durch dieses System nicht möglich, weshalb die pseudonymen Daten nicht der Erklärungspflicht personenbezogener Daten der DSGVO unterliegen.
  • Da diesem System keinerlei personenbezogene Daten vorliegen, kann auch keine Datenauskunft gem. DSGVO erteilt werden, da der Personenbezug von Daten nicht ermittelt werden kann.
  • Sofern - regelwidrig - doch personenbezogene Daten an dieses System zur Speicherung übermittelt wurden, ist dem Nutzer dieser Vorgang inkl. Daten bereits vorher bekannt. In diesem Fall sieht die DSGVO dann keine Datenauskunft vor.
  • Außerdem sind sowieso alle gespeicherten Daten dieses Systems öffentlich und jederzeit frei zugänglich. Die Datenauskunft kann durch den Download der gesamten Datenbank hier leicht selbst durchgeführt werden. Die Daten sind maschinenlesbar und selbsterklärend organisiert.
  • Der Download ist ein "git"-Repository, das entsprechende JSON-Dateien enthält.
  • Das Recht auf Vergessen wird durch Unterdrückung der Daten (siehe "Report"- und "Block"-Button) umgesetzt.
  • Dieses System verfügt weder über einen eigenen Login, noch verwendet es permanente (gespeicherte) Cookies.
  • JavaScript ist auf dieser Seite erforderlich, da die Umsetzung anders technisch schwierig würde. Nur mit JavaScript können Hashes auf der Clientseite erzeugt werden.
  • Das Session-Cookie ist aus Sicherheitsgründen notwendig, um auf Logins verzichten zu können und die Session vom URL fernzuhalten.
  • Das Session-Cookie verschwindet in der Regel, indem man die Webseite vollständig schließt und wieder öffenet.
  • Diese Webseite funktioniert problemlos über Anonymizer sowie im privaten Modus vom Browser.
  • Dieses System verwendet SSL.
  • SSL ist in sofern datenschutztechnisch eine Gefahr, weil der Browser das Surfverhalten über die OCSP-Anfragen prinzipiell trackbar macht.
  • Aus diesem Grund wird OCSP-Stapeling eingesetzt, ein Verfahren, dass die problematischen OCSP-Anfragen in die Cloud oftmals vermeiden kann.
  • Allerdings funktioniert das leider nicht immer, denn OCSP ist ein geradezu fahrlässig fragiles Konstrukt.
  • Ich bitte dafür um Verständnis, denn technisch lässt sich das nur von den Browserherstellen und dem W3C lösen, nur verstehen die in Sachen Datenschutz leider nur amerikanisch.
